Technical Specifications

Architectural constraints and platform attributes for StackPilot deployment.

Data Ingest Policy Zero-Ingest (Metadata namespaces only, ARNs, VPC routing tables, resource counts)
Permissions Scope Least Privilege, Read-Only IAM delegation (SecurityAudit + Custom Policy)
Sync Mechanism Agentless discovery querying standard cloud and cluster APIs
Scaling Latency < 2.0s discovery overhead per 10k managed resources
Supported Environments AWS (Global & Opt-In Regions), EKS, self-managed Kubernetes v1.22+
Licensing Model Permissive Apache-2.0 License (Open-Source code, schemas, and Helm charts)

EVALUATE STACKPILOT

Get Started in 60 Seconds

StackPilot is open source and self-hostable. Run it locally or deploy it to your Kubernetes cluster.

Option 1: Docker Compose

Perfect for local evaluation and testing discovery flows.

bash 
# Clone the repository
git clone https://github.com/siliconroots/stackpilot.git
cd stackpilot

# Boot up the local database and platform services
docker-compose up -d

# Open StackPilot Console in your browser
echo "http://localhost:3000"

Option 2: Kubernetes (Helm)

Deploy a scalable instance inside your EKS/K8s infrastructure.

bash 
# Add the Helm repository
helm repo add siliconroots https://charts.siliconroots.io
helm repo update

# Install StackPilot under the platform namespace
helm install stackpilot siliconroots/stackpilot \
  --namespace platform \
  --create-namespace \
  --values values.yaml
