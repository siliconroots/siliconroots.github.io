Technical Specifications

Architectural constraints and platform attributes for StackPilot deployment.

Data Ingest Policy Zero-Ingest (Metadata namespaces only, ARNs, VPC routing tables, resource counts)
Permissions Scope Least Privilege, Read-Only IAM delegation (SecurityAudit + Custom Policy)
Sync Mechanism Agentless discovery querying standard cloud and cluster APIs
Scaling Latency < 2.0s discovery overhead per 10k managed resources
Supported Environments AWS (Global & Opt-In Regions), EKS, self-managed Kubernetes v1.22+
Licensing Model Permissive Apache-2.0 License (Open-Source code, schemas, and Helm charts)

FAQ

Frequently Asked Questions

Straight answers about Silicon Roots and StackPilot.

What is Silicon Roots?

Silicon Roots is an engineering organization building open-source tools for DevOps, SRE, platform, and cloud teams. StackPilot—our cloud intelligence platform for AWS—is our first project, with more tools in development. We build in the open and welcome new ideas and collaborators.

What is StackPilot?

StackPilot is an agentless cloud intelligence platform for AWS. It discovers resources across accounts and regions, builds a unified inventory, maps topology and dependencies, and surfaces security, FinOps, and operational insights through a single console.

Why was StackPilot built?

Cloud environments grow faster than the tools used to understand them. Context ends up scattered across consoles, accounts, and tribal knowledge. StackPilot exists to make infrastructure easier to understand than it is to create.

Who is StackPilot for?

Platform, DevOps, Cloud, and Infrastructure teams operating complex AWS estates. Security teams use it to track compliance drift, FinOps teams to audit cost, and engineering leaders to keep visibility over system architecture.

Is StackPilot open source?

Yes. StackPilot is open source under the permissive Apache-2.0 license—code, schemas, and Helm charts. You can self-host it locally or in your own Kubernetes cluster, and read exactly how it works.

How can I contribute?

Open an issue, file a feature request, start a discussion, or submit a pull request on GitHub. Operator feedback directly shapes the roadmap, and contributions are reviewed in the open.

How does AWS onboarding work?

You connect an AWS account using a read-only IAM cross-account role (SecurityAudit plus a small custom policy). StackPilot validates the connection and runs agentless discovery against standard cloud APIs—no agents, sidecars, or write access required.

How are credentials protected?

StackPilot requests least-privilege, read-only access only and never stores write credentials. It indexes metadata namespaces only—ARNs, resource types, VPC routing tables—and never reads database rows, file contents, or application payloads.

What is the roadmap?

StackPilot for AWS is the active focus. Next come InfraGraph and a Policy Engine, followed by a Cloud SDK, multi-cloud (Azure and GCP) intelligence, and AI-assisted insights. See the Future Direction section above for the full timeline.

What future projects are planned?

InfraGraph (a versioned cloud relationship graph), a Policy Engine (OPA/Rego compliance as code), and a Cloud SDK & CLI for programmatic access. Beyond the cloud intelligence suite, Silicon Roots is also exploring dedicated DevOps and SRE tooling—covering incident workflows, deployment automation, and reliability engineering. These are long-term future scope items and are clearly marked as such on the roadmap.
